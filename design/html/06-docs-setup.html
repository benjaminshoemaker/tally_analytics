Setting up Tally Analytics

Follow this step-by-step guide to integrate privacy-friendly analytics into your Next.js application using our automated GitHub bot.

schedule Estimated read time: 5 min
1

Prerequisites

Before we begin, please ensure your project meets the following requirements:

  • check_circle Next.js version 13.4 or later (App Router recommended)
  • check_circle A GitHub repository connected to your Vercel deployment
  • check_circle Admin access to the GitHub repository
2

Install GitHub App

Tally Analytics uses a GitHub App to automatically configure your project. This ensures that your analytics setup is always up-to-date with our latest privacy features.

Step 2.1: Authorize the App

Click the button below to redirect to GitHub and select the repository you want to instrument.

Install Tally on GitHub

Screenshot: GitHub Permissions Screen

3

Review Pull Request

Once installed, Tally will automatically open a Pull Request in your repository. This PR includes a configuration file at tally.config.ts. You can customize the analytics behavior here.

info

Important

The default configuration is set to "Strict Privacy". This means no cookies are stored on the user's device by default.

tally.config.ts 
import { defineConfig } from '@tally/core';

export default defineConfig({
  // Your unique project ID
  projectId: 'pj_283849102',
  
  // Privacy settings
  privacy: {
    mode: 'strict', // 'strict' | 'standard' | 'loose'
    ignoreLocalhost: true,
  },
  
  // Optional: Exclude specific paths
  exclude: [
    '/admin/**',
    '/api/**'
  ]
});

Merge the Pull Request to deploy the configuration to your main branch. Vercel will trigger a redeploy automatically.

4

Verify Installation

Visit your deployed application and navigate through a few pages. Then, return to the Tally Dashboard.

Open Dashboard open_in_new It may take up to 2 minutes for the first event to appear.
